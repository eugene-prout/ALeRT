A grammar is a specification for a set of valid strings in a language. These grammars help describe the structure of a language in a logical way, making them useful for building compilers.

The First Set is the property of a grammar (Or a single grammar symbol), describing the terminals that can appear as the first symbol derived from another symbol, typically a non-terminal.

This is a useful concept as the First Set can help us craft a grammar that is equivalent to the one we want, but with less of some undesirable properties like Left Recursion.

Left Recursion is when a non-terminal in a grammar can derive itself as the first symbol. This can be either direct (A ::= A) or indirect through a chain of rules. There are algorithmic ways to eliminate this left-recursion, which allows top-down parsers to parse the grammar.

This often results in a less intuitive grammar, and it can hurt associativity when parsing, requiring extra work.

First Sets of non-terminals have a dependency relationship between each other. For example in A ::= B, the first set of A includes that of B, resulting in a 'dependency'.

ALeRT generates a graph of these dependencies to help visualise where to look in a grammar to eliminate left recursion.
